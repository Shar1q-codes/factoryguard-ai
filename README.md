# FactoryGuard AI — Predictive Maintenance System

> Predicts turbofan engine failure probability from real-time sensor data.  
> Built on NASA CMAPSS FD001 dataset · XGBoost · SHAP Explainability · Flask REST API

---

## What This Project Does

Industrial machines fail unexpectedly — causing downtime, safety risks, and huge costs.  
FactoryGuard AI solves this by analyzing 21 sensor readings from a turbofan engine and predicting **whether the engine will fail within the next 30 cycles** — before it actually happens.

A factory operator can send live sensor readings to our REST API and instantly receive a **failure probability score** (0.0 to 1.0), allowing maintenance to be scheduled proactively.

- Trains XGBoost with class imbalance correction
- Saves `data/features_df.pkl` and `data/final_model.pkl`

## How It Works

### 1. Data Pipeline (`src/data_pipeline.py`)

Loads the raw NASA CMAPSS dataset.  
Computes **Remaining Useful Life (RUL)** for each engine cycle.  
Creates a binary label `failure = 1` when RUL ≤ 30 cycles, else `failure = 0`.

### 2. Feature Engineering (`src/feature_engineering.py`)

Raw sensor readings alone are weak predictors — the trend matters more than the value.  
We engineer **189 rolling features** and **42 lag features** per row:

| Feature Type | Description | Count |
|---|---|---|
| Rolling mean | Average sensor value over last N cycles | 63 |
| Rolling std | Variability over last N cycles | 63 |
| EMA | Exponential moving average (recent cycles weighted more) | 63 |
| Lag-1 | Sensor value 1 cycle ago | 21 |
| Lag-2 | Sensor value 2 cycles ago | 21 |

All features are computed **within each machine unit** using `groupby('unit_nr')` — readings from different engines never mix.

Final feature matrix: **24,640 rows × 256 features**

### 3. Model Training (`src/train.py`)

Algorithm: **XGBoost** (gradient boosted trees)  
Class imbalance handled via `scale_pos_weight = negatives / positives = 6.85`  
Train/test split: 80/20 by machine unit (not by row shuffle)

| Model | PR-AUC |
|---|---|
| XGBoost | **1.0000** |
| LightGBM | 1.0000 |

Primary metric: **PR-AUC** (Precision-Recall AUC) — chosen because the dataset is imbalanced (~13% positive class). Accuracy would be misleading here.

### 4. SHAP Explainability (`reports/plots/`)

SHAP (SHapley Additive exPlanations) shows **which sensors matter most** for each prediction.

- `shap_summary.png` — feature importance across all predictions
- `shap_waterfall.png` — how each feature pushed one prediction up or down
- `shap_bar.png` — mean absolute SHAP values ranked by importance

### 5. Flask API (`api/app.py`)

Single endpoint: `POST /predict`  
Input: JSON with sensor readings  
Output: `failure_probability` (float 0–1) + `status` label  
Latency: under 50ms per request

## Troubleshooting

**`FileNotFoundError: data/train_FD001_with_RUL.csv`**  
→ Dataset file is missing. Place it in the `data/` folder (see Step 3).

**`ModuleNotFoundError`**  
→ Dependencies not installed. Run `pip install -r requirements.txt` (Step 2).

**`ConnectionRefusedError` when calling the API**  
→ Flask server is not running. Start it with `python api/app.py` (Step 5).

**Port 5000 already in use**  
→ Kill the process using it:
```bash
# Mac / Linux
lsof -ti:5000 | xargs kill -9

# Windows
netstat -ano | findstr :5000
taskkill /PID <PID> /F
```
